Как узнать сетевой интерфейс контейнера на Хосте

Есть контейнер с ID d31aeb60a60fsdfg:

$ crictl ps # В моей системе установлен containerd
CONTAINER           IMAGE               CREATED             STATE               NAME        ...
d31aeb60a60fsd      55f4b40fe486a       14 minutes ago      Running             app         ...

Задача узнать его хостовой интерфейс.

  1. В контейнере есть sh:
$ crictl exec -it d31aeb60a60fsd sh
$ cat /sys/class/net/eth0
46
$ exit
$ ip a | grep ^46
46: calidf173e47a38@if4: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1440 qdisc noqueue state UP group default
  1. В контейнере отсутствует sh:
$ crictl inspect --output go-template --template '{{.info.pid}}' d31aeb60a60fsd
1659
### Если у вас docker: docker inspect -f '{{.State.Pid}}' <container id>
$ ip netns identify 1659
cni-32258e36-761a-7563-5227-d53dd933058b
$ ip netns exec cni-32258e36-761a-7563-5227-d53dd933058b cat /sys/class/net/eth0/iflink
46
$ ip a | grep ^46
46: calidf173e47a38@if4: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1440 qdisc noqueue state UP group default

Документация по /sys/class/net

Документация по ip-netns(8)

Автоматизация пункта №2 #

Листинг скрипта:

#!/bin/bash

CONTAINER_ID=$1
CONTAINER_DIR='/run/containerd/io.containerd.runtime.v2.task/k8s.io'

if ! CONTAINER_PID=$(cat $CONTAINER_DIR/$CONTAINER_ID/init.pid); then
    exit 1
fi

NET_NS_ID=$(ip netns identify $CONTAINER_PID)
INTERFACE_IFLINK=$(ip netns exec $NET_NS_ID cat /sys/class/net/eth0/iflink)
INTERFACE_NAME=$(ip a | grep ^$INTERFACE_IFLINK)

echo $INTERFACE_NAME | awk -F": " '{print $2}' | awk -F "@" '{print $1}'

Потребуется передать в скрипт полный ID контейнера, без усечений:

$ crictl ps --no-trunc
CONTAINER                                                          IMAGE                            
d31aeb60a60f35bcb48c044fe683713f6e917572954e12139ac3688efdae8acc   sha256:5d725196c1f49aa3663ac...

Запустим скрипт

$ ./script.sh d31aeb60a60f35bcb48c044fe683713f6e917572954e12139ac3688efdae8acc
calidf173e47a38
