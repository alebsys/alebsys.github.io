Performance kube-dns

Проблема #

В нашей инсталяции при дефолтных настройках core-dns не эффективно обрабатывает запросы:

  • лишние попытки разрешения с суффиксами из секции search в /etc/resolve.conf
  • резолв AAAA записей

Что в итоге приводит к росту latency всей системы и в целом перегружает Поды лишней работой.

Перечислю основные подстройки для улучшения ситуации.

Отключение резолва АААА записей #

Вне зависимости от того используются ли ipv6 адреса в кластере или нет по дефолту core-dns будет пытаться разрешать и AAAA записи в том числе.

Чтобы отключить резолв AAAA записей следует добавить в конфиг:

template ANY AAAA {
    rcode NXDOMAIN
}

p.s. плагин template включен по умолчанию.

Указание имен как FQDN #

Как было описано в соседнем топике, величина ndots определяет когда имя будет считаться FQDN. Поэтому при стандартных ndots:5 домен example.com будет резолвиться только с четвертой попытки.

Для улушения ситуации можно явно задавать . в конце url: example.com. , тогда example.com будет резолвиться с первой попытки.

NodeLocal DNSCache #

Для ускорения трансляции имен возможно использовать локальный DNS кеш - NodeLocal DNSCache. С версии kubernetes 1.18 он вышел в stable mode.

NodeLocal DNSCache выступает прослойкой между клиентом и сервером DNS, располагаясь на каждой машине кластера как экземпляр daemonSet.

  1. Клиент посылает DNS запрос в NodeLocal DNSCache;
  2. NodeLocal DNSCache отвечает клиенту если ответ был закеширован, иначе
  3. NodeLocal DNSCache обращается в core-dns для обновления своих данных;
  4. NodeLocal DNSCache отвечает клиенту.

Таким образом при “удачном” обращении запрос не покидает Ноду с Подом клиента - избегает DNAT преобразований, походов по сети и как следствие разгружает core-dns Поды.

dnsConfig per Pod #

Если известно, что приложение в Поде ходит только вовне, то на уровне секции spec.dnsConfig манифеста возможно кастомно выставить параметр ndots :

apiVersion: v1
kind: Pod
metadata:
  namespace: default
  name: dns-example
spec:
...
  dnsConfig:
    options:
      - name: ndots
        value: "2"

Игры с ndots #

Саму настройку ndots так же возможно изменять, подстраивая значение под глубину поддоменов принятых в организации.
