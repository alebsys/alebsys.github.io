REST API на golang. Часть первая

Преамбула #

На YouTube мне попался толковый курс Building Microservices with Go от Nic Jackson. Параллельно с изучением буду писать небольшие заметки.

Часть первая #

В первой части познакомимся с азами http пакета и построим простой, но вполне рабочий вебсервер.

Примером выступит:

package main

import (
	"fmt"
	"io/ioutil"
	"log"
	"net/http"
)

func handlerHello(rw http.ResponseWriter, r *http.Request) {
	log.Println("request to /hello")
	d, err := ioutil.ReadAll(r.Body)
	if err != nil {
		http.Error(rw, "error!", http.StatusBadRequest)
		return
	}

	fmt.Fprintf(rw, "Hello %s", d)
}

func handlerBye(rw http.ResponseWriter, r *http.Request) {
	log.Println("request to /bye")
	d, err := ioutil.ReadAll(r.Body)
	if err != nil {
		http.Error(rw, "error!", http.StatusBadGateway)
		return
	}

	fmt.Fprintf(rw, "Bye %s", d)
}

func main() {

	http.HandleFunc("/hello", handlerHello)
	http.HandleFunc("/bye", handlerBye)

	http.ListenAndServe(":9090", nil)
}

Приступим.

Точка входа. ListenAndServe() #

Начнем с определения функции:

func http.ListenAndServe(addr string, handler http.Handler) error

ListenAndServe() принимает в качестве параметров addr и обработчик handler, где:

  • addr - адрес и порт которые “слушает” вебсервер;
  • handler - обработчик, чья задача, как следует из названия, обрабатывать входящие запросы.

В нашем случае:

http.ListenAndServe(":9090", nil)
  • Веб-сервер ожидает подключений на 9090 порту с любых адресов;
  • Конкретный handler не определен (nil), поэтому будет использоваться стандартный - DefaultServeMux. В будущем коснемся этой темы.

Что есть Handler #

Handler под капотом являет собой тип interface:

type Handler interface {
	ServeHTTP(ResponseWriter, *Request)
}

Метод ServeHTTP() принимает Request (структура) и пишет ответ в ResponseWriter (так же интерфейс).

В целом логика следующая:

  1. поступает новый запрос по url <SERVER_ADDRESS>:9090/hello ;
  2. ListenAndServe() вызывает обработчик handlerHello, так как путь /hello совпадает с его pattern.

Регистрация handler #

Перед использованием, следует зарегистрировать обработчик через функцию HandleFunc():

func http.HandleFunc(pattern string, handler func(http.ResponseWriter, *http.Request))
  • pattern - проверяет соответствие пути запроса. Подробнее в Wiki;
  • handler - определенный экземпляр обработчика.

Подобных функций может быть сколь угодно много. Таким образом для обработки запроса будет вызываться тот handler чей pattern будет наиболее близок к значению пути в url.

В нашей программе объявлено два handler‘а:

http.HandleFunc("/hello", handlerHello)
http.HandleFunc("/bye", handlerBye)

Поэтому запрос на <SERVER_ADDRESS>:9090/hello обработается handlerHello, а handlerBye возьмется за <SERVER_ADDRESS>:9090/bye.

Подробнее про сопоставление пути в url с pattern читай в документации к пакету.

Запрос - ответ #

Пробежимся по реализации handlerHello():

func handlerHello(rw http.ResponseWriter, r *http.Request) {
	log.Println("request to /hello")
	d, err := ioutil.ReadAll(r.Body)
	if err != nil {
		http.Error(rw, "error!", http.StatusBadRequest)
		return
	}

	fmt.Fprintf(rw, "Hello %s", d)
}
  1. Логируем поступление запроса на обработчик;
  2. Пишем в переменную d тело запроса;
  3. Проверяем ошибки в шаге (2). При наличии пишем в rw, читай отправляем клиенту, код ошибки 400 и выходим из функции;
  4. Если ошибок нет, отправляем в rw ответ.

Стоит заметить, что функция fmt.Fprintf() принимает первым параметром интерфейс w io.Writer. И так как http.ResponseWriter реализует метод Write(), то мы вполне успешно можем подставить его в функцию.

Запустим наш код и отправим запросы к веб-серверу:

curl 127.0.0.1:9090/hello -d 'World' # отправляем запрос на /hello
Hello World # получаем ответ от веб-сервера
curl 127.0.0.1:9090/bye -d 'World' # отправляем запрос на /bye
Bye World # получаем ответ от веб-сервера
